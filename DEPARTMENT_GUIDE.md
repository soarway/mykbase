# 部门管理页面使用指南

## 📋 页面概述

部门管理页面采用**左右分栏**布局，完美还原您的设计草图：
- **左侧（55%）**：树形结构的部门列表
- **右侧（45%）**：部门详情编辑表单

## 🎨 页面特性

### 🌲 左侧 - 部门树

#### 1️⃣ **操作按钮区**
```
┌─────────────────────────┐
│ [添加根部门] [添加下级] │
└─────────────────────────┘
```
- **添加根部门**：蓝色按钮，创建顶级部门
- **添加下级**：白色边框按钮，为选中部门添加子部门

#### 2️⃣ **搜索框**
- 按部门名称搜索
- 实时过滤部门列表
- 清晰的搜索图标提示

#### 3️⃣ **树形结构**
```
▼ 上海嘉成
  ├─ 研发部
  ├─ 财务部
  └─ 智慧轨道

▼ 上海技服
  ├─ 行政部
  ├─ 客服部本部
  ├─ ▶ 工程技术部
  │   └─ 财务部
  └─ ▶ 总裁办
```

**交互功能：**
- ✅ 点击展开/收起图标：控制子部门显示
- ✅ 点击部门名称：选中并加载到右侧编辑
- ✅ 层级缩进显示
- ✅ 选中高亮（蓝色背景）
- ✅ 悬停效果（浅灰背景）

#### 4️⃣ **树形节点图标**
- `▼` - 已展开的父部门
- `▶` - 已收起的父部门
- 🏢 - 所有部门的图标
- 缩进表示层级关系

### 📝 右侧 - 编辑表单

#### 1️⃣ **标签页**
```
基本编辑
───────
```
- 当前只有一个标签
- 可扩展添加更多标签（如：权限设置、成员管理等）

#### 2️⃣ **表单字段**

##### **机构名称*** （必填）
- 文本输入框
- 带清除按钮（×）
- 红色星号标识必填

##### **上级部门**
- 下拉选择框
- 选项：所有部门列表
- 可选"无（根部门）"
- 自动排除当前部门（防止循环引用）

##### **机构编码**
- 文本输入框
- 用于系统内部标识
- 示例：A01A05

##### **排序**
- 数字输入框
- 控制同级部门的显示顺序
- 数字越小越靠前

#### 3️⃣ **操作按钮**
```
┌──────────────────┐
│  [重置]  [保存]  │
└──────────────────┘
```
- **重置**：清空表单，取消选中
- **保存**：提交部门信息（演示版显示提示）

#### 4️⃣ **空状态**
未选中任何部门时显示：
```
    🏢
请从左侧选择一个部门进行编辑
```

## 🎯 使用流程

### 场景1：查看部门信息
1. 在左侧树中点击部门名称
2. 右侧自动加载部门详情
3. 查看各项信息

### 场景2：编辑部门信息
1. 点击选择要编辑的部门
2. 右侧表单自动填充数据
3. 修改相应字段
4. 点击"保存"提交

### 场景3：重置表单
1. 点击"重置"按钮
2. 表单清空
3. 取消部门选中状态

### 场景4：搜索部门
1. 在搜索框输入部门名称
2. 树形列表实时过滤
3. 点击搜索结果查看详情

## 💾 数据结构

```typescript
interface Department {
  id: string;           // 部门ID
  name: string;         // 部门名称
  code: string;         // 机构编码
  parentId: string | null;  // 上级部门ID（null表示根部门）
  sort: number;         // 排序号
  children?: Department[];  // 子部门列表
}
```

## 🔌 API对接建议

### 1️⃣ 获取部门树
```typescript
// GET /api/departments/tree
interface DepartmentTreeResponse {
  success: boolean;
  data: Department[];
}
```

### 2️⃣ 创建部门
```typescript
// POST /api/departments
interface CreateDepartmentRequest {
  name: string;
  code: string;
  parentId?: string;
  sort: number;
}
```

### 3️⃣ 更新部门
```typescript
// PUT /api/departments/:id
interface UpdateDepartmentRequest {
  name: string;
  code: string;
  parentId?: string;
  sort: number;
}
```

### 4️⃣ 删除部门
```typescript
// DELETE /api/departments/:id
// 注意：需要处理有子部门的情况
```

### 5️⃣ 搜索部门
```typescript
// GET /api/departments/search?keyword=xxx
interface SearchDepartmentResponse {
  success: boolean;
  data: Department[];

```

## 📱 响应式设计

- **桌面端**：左右分栏，比例 55:45
- **未来扩展**：移动端可改为上下布局或抽屉式

## 🔒 权限建议

建议的权限控制点：
- 查看部门树：所有登录用户
- 添加/编辑部门：管理员
- 删除部门：超级管理员
- 部门搜索：所有登录用户

## 📊 数据验证

### 前端验证
- ✅ 机构名称不能为空
- ✅ 机构编码格式验证
- ✅ 上级部门不能选择自己
- ✅ 排序号必须是数字

### 后端验证（建议）
- 部门名称唯一性（同级）
- 机构编码唯一性（全局）
- 防止循环引用
- 删除前检查是否有子部门
- 删除前检查是否有关联用户
